# 設計（Design）

## 使用技術

| 種別 | 技術 |
|------|------|
| フレームワーク | Next.js（App Router） |
| 言語 | TypeScript |
| スタイリング | Tailwind CSS |

---

## Phase 1 フォルダ構成（現状・完了）

```
app/
  layout.tsx              # 共通レイアウト
  page.tsx                # 育休給付金シミュレーター（/）
  legal/page.tsx          # 利用規約・プライバシーポリシー
  components/             # シミュレーター用コンポーネント
  lib/                    # 計算ロジック・型定義
```

---

## Phase 2 フォルダ構成（ハブ化後）

```
app/
  layout.tsx              # 共通レイアウト（全ページ共通のHTML骨格）
  page.tsx                # ハブトップページ（新規作成）
  ikukyu/
    page.tsx              # 育休給付金シミュレーター（app/page.tsx から移動）
  legal/
    page.tsx              # 変更なし
  components/             # 変更なし（コンポーネントは ikukyu から引き続き参照）
  lib/                    # 変更なし
```

コンポーネントは `app/components/` のままで問題ない。
将来 `/kakeibo` 等を追加する際は `app/kakeibo/page.tsx` を作るだけで `/kakeibo` が生える。

---

## ハブトップページ（app/page.tsx）の構成

```
┌─────────────────────────────┐
│  サイト名・キャッチコピー         │  ← "出産・育休 お金サービス" 系の名前（要決定）
├─────────────────────────────┤
│  ツールカード（1枚目）            │
│  🧮 育休給付金シミュレーター       │  ← /ikukyu へリンク
│  産後からもらえる給付金が全部わかる  │
├─────────────────────────────┤
│  （将来のツールカードはここに追加）  │
└─────────────────────────────┘
```

シンプルに「ツール一覧ページ」として始める。
記事・ブログ等は後フェーズで追加。

---

## 変更が必要なリンク

| ファイル | 変更内容 |
|---------|---------|
| `app/components/Footer.tsx` | © の文言・シミュレーターリンクを `/ikukyu` に変更 |
| `app/legal/page.tsx` | `← シミュレーターへ` のリンクを `/` → `/ikukyu` に変更 |
| `app/ikukyu/page.tsx` | 特になし（計算機はそのまま移動） |

---

## メタ情報（metadata）の設計

現在 `app/layout.tsx` にサイト全体の metadata が1つあるだけ。
ページごとに異なる title / description を設定するため、各ページに `export const metadata` を追加する。

| ページ | title | description |
|-------|-------|-------------|
| `/`（ハブ） | サイト名（要決定） | サイト全体の説明 |
| `/ikukyu` | 育休給付金シミュレーター | 産後から育休まで、もらえるお金が全部わかる |
| `/legal` | 利用規約・プライバシーポリシー | （簡易な説明） |

---

## 決定事項（Phase 2 実装時に確定）

1. **ハブのサイト名**: 「ikukyu」に決定
2. **リダイレクト**: 未インデックスのためリダイレクトなし（方針A）
3. **Footer**: 全ページ共通（app/layout.tsx 経由）のまま維持
